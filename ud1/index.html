Concurrencia en Java

La concurrencia permite ejecutar múltiples tareas de forma solapada, ya sea en un único núcleo mediante cambio de contexto o en varios núcleos en paralelo. En Java, la base son los hilos (Thread) y el paquete java.util.concurrent, que ofrecen executors, futures, colas bloqueantes, sincronizadores y utilidades de alto nivel.

  • Modelo: cada proceso de la JVM puede crear múltiples hilos que comparten memoria.
  • Buenas prácticas: evitar condiciones de carrera, usar inmutabilidad, contenedores concurrentes y executors en lugar de crear hilos a mano.
  • Bloqueos y coordinación: synchronized, Lock, Semaphore, CountDownLatch, CompletableFuture, etc.

En esta guía nos centramos en cómo la JVM interactúa con el sistema operativo para lanzar procesos externos, medirlos, redirigir su E/S y gestionarlos de forma segura.

Mapa mental rápido
  • Runtime = puerta a recursos del entorno de la JVM.
  • ProcessBuilder = configurar y lanzar procesos nativos.
  • Process = controlar un proceso ya en ejecución.
  • E/S: getInputStream(), inheritIO(), redirecciones.
  • Control: waitFor(), isAlive(), destroy().

Clase Runtime

Docs oficiales

Runtime es un singleton accesible con Runtime.getRuntime(). Expone operaciones para interactuar con el entorno donde corre la JVM: memoria, procesos externos, ganchos de apagado, etc.

Método Funcionalidad básica Notas
getRuntime() Obtiene la instancia única de Runtime. Patrón singleton.
availableProcessors() Número de CPU lógicas disponibles para la JVM. Útil para dimensionar thread pools.
freeMemory(), totalMemory(), maxMemory() Estadísticas de memoria del heap. Para diagnosticar consumo y límites del heap.
gc() Solicitud (no garantizada) de recolección de basura. No fuerces GC en producción salvo diagnóstico.
exec(...) Lanza un proceso nativo y devuelve Process. Se prefiere ProcessBuilder por claridad/flexibilidad.
addShutdownHook(Thread) Registra tareas a ejecutar al apagarse la JVM. Para liberar recursos, cerrar logs, etc.
halt(int) Termina la JVM inmediatamente. Uso excepcional; no ejecuta shutdown hooks.

Clase Process

Docs oficiales

Process representa y controla un proceso nativo iniciado por ProcessBuilder.start() o Runtime.exec(). Permite leer su salida, escribir en su entrada, esperar a que termine, consultar su código de salida y finalizarlo.

Método ¿Para qué sirve? Notas de uso
getInputStream(), getErrorStream(), getOutputStream() Leer STDOUT/STDERR del proceso y escribir en STDIN. Consume la salida para evitar bloqueos por buffers.
waitFor(), waitFor(timeout, unit) Bloquea hasta que termine (con o sin tiempo máximo). Combínalo con lectura en hilos/async si hay mucha salida.
isAlive(), exitValue() Estado de vida y código de salida si ya terminó. exitValue() lanza excepción si sigue vivo.
destroy(), destroyForcibly() Solicita terminar el proceso (suave / forzado). Usa forzado solo si no responde al cierre normal.
pid(), info() Identificador del proceso y metadatos (cmd, tiempos, etc.). info() devuelve ProcessHandle.Info.

Clase ProcessBuilder

Docs oficiales Guía: Crear procesos

ProcessBuilder configura los atributos (comando, directorio de trabajo, entorno, redirecciones) para lanzar uno o varios procesos. Con start() obtienes un Process listo para controlar.

Método / atributo Funcionalidad Ejemplo/Comentarios
new ProcessBuilder(cmd...) / command() Define el ejecutable y sus argumentos. new ProcessBuilder("ls","-la")
directory(File) Establece el directorio de trabajo del proceso. pb.directory(new File("/tmp"))
environment() Mapa mutable de variables de entorno. pb.environment().put("MODE","dev")
redirectErrorStream(boolean) Fusiona STDERR en STDOUT. Simplifica la lectura secuencial de mensajes.
redirectInput/Output/Error(...) y Redirect Redirige E/S a ficheros, hereda o descarta. Redirect.INHERIT, Redirect.DISCARD, Redirect.to(file)
inheritIO() El proceso usa la misma consola que la JVM. Útil para ver la salida “en vivo”.
start() Crea y arranca el proceso con los atributos definidos. Devuelve Process.
startPipeline(List<ProcessBuilder>) Encadena procesos conectando su E/S (tuberías). Ej.: grep | sort | uniq-like en Java.
Ejemplo mínimo
ProcessBuilder pb = new ProcessBuilder("echo", "Hola mundo");
pb.redirectErrorStream(true).inheritIO();
Process p = pb.start();
int code = p.waitFor();
